📸 Диаграмма последовательности (Sequence Diagram) по ГОСТу. Показывает не только сам протокол OAuth 2.0, но и то, как токен передаётся из React в Django, а затем Celery запускает фоновую загрузку тренировок. Делай скриншот для Рисунка 3.3.
sequenceDiagram autonumber actor User as Пользователь participant React as Frontend (React) participant Django as Backend (Django) participant DB as PostgreSQL participant Celery as Celery Worker participant StravaAuth as Strava Auth API participant StravaData as Strava Data API User->>React: Клик "Подключить Strava" React->>StravaAuth: Redirect (client_id, scope, redirect_uri) StravaAuth-->>User: Страница авторизации Strava User->>StravaAuth: Ввод учетных данных, выдача прав StravaAuth-->>React: Redirect обратно с authorization_code React->>Django: POST /api/integrations/strava/exchange/ (code) Django->>StravaAuth: POST /oauth/token (code, client_secret) StravaAuth-->>Django: Возврат access_token, refresh_token Django->>DB: Сохранение токенов в integrations_stravaconnection Django->>Celery: Вызов фоновой задачи sync_activities.delay() Django-->>React: 200 OK (Токены сохранены) React-->>User: Статус "Синхронизация в фоне..." rect rgb(240, 248, 255) Note over Celery,StravaData: Процесс первичной синхронизации (Асинхронно) Celery->>DB: Чтение access_token Celery->>StravaData: GET /athlete/activities?per_page=200 StravaData-->>Celery: JSON (список активностей) Celery->>DB: Пакетное сохранение в activities_activity loop Для каждой новой активности Celery->>StravaData: GET /activities/{id}/streams StravaData-->>Celery: JSON (координаты, пульс) Celery->>DB: Сохранение в activities_stream end end
