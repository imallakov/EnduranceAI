# Речь для защиты ВКР

**Советы перед выступлением:**
1. **Не читай с листа:** Распечатай этот текст, но старайся рассказывать своими словами, лишь подглядывая в него.
2. **Тайминг:** Текст рассчитан примерно на 7-8 минут спокойного выступления (это стандарт для защиты ВКР). 
3. **Управление вниманием:** Когда дойдешь до Слайда 12 (Результаты), сделай паузу перед цифрой 52% и скажи её громче — это твой главный козырь.

---

### Слайд 1. Титульный (15 сек)
«Уважаемый председатель, уважаемые члены государственной экзаменационной комиссии! Вашему вниманию представляется выпускная квалификационная работа на тему: "Разработка автоматизированной системы предиктивной аналитики и адаптивного планирования тренировочного процесса EnduranceAI". Работу выполнил студент Аллаков Якуп».

### Слайд 2. Актуальность темы (30 сек)
«Актуальность темы обусловлена взрывным ростом массового любительского спорта. Аудитория только одного приложения Strava сегодня превышает 125 миллионов пользователей. Однако существующие массовые платформы прогнозируют результаты марафонов по максимально упрощенным формулам, полностью игнорируя рельеф трассы и погодные условия. При этом на рынке существует огромный разрыв между академической спортивной наукой и реальным пользовательским опытом в мобильных приложениях».

### Слайд 3. Цель и задачи (30 сек)
«Целью данной работы является разработка информационной системы EnduranceAI, которая закрывает эту потребность за счёт гибридной прогнозной модели и адаптивного генератора планов. 
Для достижения цели были решены следующие задачи: проанализированы существующие решения, спроектирована архитектура, обучена модель машинного обучения и реализовано полноценное веб-приложение для конечных пользователей».

### Слайд 4. Объект, предмет и методы (20 сек)
*(Комиссия сама может прочитать этот слайд, поэтому говорим коротко)*
«Объектом исследования выступает процесс подготовки спортсмена-любителя к марафону. Предметом — методы и алгоритмы автоматизированной поддержки принятия решений. В работе применялись методы системного и объектно-ориентированного проектирования, а также градиентный бустинг и методы математической статистики».

### Слайд 5. Анализ существующих решений (30 сек)
«В ходе сравнительного анализа таких платформ как Strava, Garmin Connect и TrainingPeaks было выявлено, что ни одна система на рынке не объединяет одновременный учёт физиологической формы атлета, профиля трассы и погодных условий. Существующие продукты либо предлагают закрытые алгоритмы с привязкой к конкретным часам, либо требуют дорогой платной подписки. Это определило нишу для разрабатываемой системы EnduranceAI».

### Слайд 6. Теоретические основы модели (40 сек)
«Теоретическим фундаментом нашей модели послужили признанные мировые физиологические концепции. Это индекс беговой формы VDOT по Джеку Дэниелсу, модель импульса тренировочной нагрузки Banister-Coggan, модель расчёта энергозатрат на градиенте от Минетти и поправки на тепловой стресс по медицинским стандартам ACSM. Финальная же корректировка прогноза осуществляется с помощью алгоритма машинного обучения XGBoost».

### Слайд 7. Архитектура системы (30 сек)
«Для реализации системы была спроектирована современная многослойная архитектура. Клиентская часть взаимодействует с REST API сервера через защищенные соединения. Для асинхронной обработки тяжелых фоновых задач, таких как парсинг GPS-треков, используется брокер сообщений Redis и очередь Celery. Также реализованы безопасные интеграции с внешними API Strava и сервисами погоды».

### Слайд 8. Модель данных (30 сек)
«Спроектированная модель данных приведена к третьей нормальной форме Кодда и включает 11 основных сущностей. Центральными элементами выступают таблицы Пользователей, Активностей, Марафонов и Тренировочных планов. Для обеспечения высокой скорости построения графиков формы, реализована контролируемая денормализация в таблице ежедневных метрик нагрузки».

### Слайд 9. Реализация серверной части (Backend) (35 сек)
«Серверная часть разработана на языке Python с использованием фреймворка Django. Реализовано 18 REST-эндпоинтов, бизнес-логика которых покрыта автотестами на 67%. Особое внимание уделено фоновой обработке: система умеет асинхронно парсить гео-треки, высотные профили и пульсовые зоны из тяжелых файлов форматов FIT, GPX и TCX, абсолютно не блокируя работу интерфейса пользователя».

### Слайд 10. Реализация клиентской части (Frontend) (30 сек)
«Клиентская часть написана на языке TypeScript с использованием библиотеки React. Для системы был разработан собственный адаптивный интерфейс, который корректно отображается на экранах любых устройств — от смартфонов до широкоформатных мониторов. Реализована мультиязычность на три языка и интегрирован бесшовный процесс авторизации через Strava OAuth 2.0».

### Слайд 11. Гибридная прогнозная модель (40 сек)
«Основа научной новизны работы — гибридный пайплайн прогнозирования. Он работает как цепочка: базовое время рассчитывается по таблицам Дэниелса, затем накладывается мультипликативная поправка на рельеф трассы, а потом поправка на погоду. На последнем этапе модель градиентного бустинга XGBoost, обученная на выборке из 30 тысяч реальных марафонов, корректирует прогноз с учетом 10 нелинейных признаков».

### Слайд 12. Результаты (Точность прогноза) (40 сек)
«Главные результаты работы представлены на слайде. Оценка на отложенной тестовой выборке из шести тысяч записей показала, что последовательное применение физиологических поправок и машинного обучения позволило кардинально снизить среднюю абсолютную ошибку прогноза. Ошибка снизилась с базовых 14 минут, которые выдает алгоритм Strava, до 6,8 минут у нашей гибридной модели. Общее улучшение точности составило более 52 процентов».

### Слайд 13. Демонстрация (25 сек)
«Разработанная система полностью функционирует и развёрнута в сети Интернет. Общий объем написанной кодовой базы превысил 14 тысяч строк. Проект опубликован в открытом доступе на GitHub. Успешно реализованы все заявленные функциональные требования, от импорта тренировок до генерации адаптивных PDF-планов».

### Слайд 14. Перспективы развития (20 сек)
«Среди перспектив дальнейшего развития проекта можно выделить: расширение модели на полумарафоны и ультра-трейлы, интеграцию с API носимых устройств напрямую без посредничества Strava, а также внедрение системы динамической корректировки плана прямо по ходу недели в реальном времени».
